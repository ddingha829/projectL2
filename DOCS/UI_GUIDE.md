# 📐 Ticgle 표준 간격 & UI 가이드라인

이 문서는 티끌(Ticgle)의 고밀도 매거진 스타일 인터페이스를 유지하기 위한 **표준 디자인 시스템**을 정의합니다. 모든 신규 개발 및 스타일 수정은 본 가이드의 수치를 기준으로 합니다.

---

## 1. 섹션 간격 시스템 (Section Spacing)
모든 섹션은 헤더(`.sectionHeader`, `.magSecHeader`)의 마진값을 통해 간격을 제어하며, 콘텐츠 컨테이너의 개별 마진은 지양합니다.

| 구분 | PC 버전 (Desktop) | 모바일 버전 (Mobile) |
| :--- | :--- | :--- |
| **섹션 간 간격 (Top)** | `35px` | `12px` |
| **제목-콘텐츠 간격 (Bottom)** | `8px` | `4px` |

> [!IMPORTANT]
> **첫 번째 섹션 예외 규칙**: 최상단 섹션(예: HOTicgle)은 상단바와의 밀착을 위해 `margin-top: 0`을 적용합니다.

---

## 2. 카드 디자인 규격 (Card Specifications)
고밀도 매거진 뷰를 위한 카드 비율 및 텍스트 규정입니다.

### 모바일 A카드 (Hero/Main)
*   **비율**: `4 : 4.2` (`aspect-ratio: 4 / 4.2`)
*   **내용**: 최대 **2줄** 제한 (말줄임표 처리)
*   **제목**: 기본 대비 **15%** 확대

### 모바일 B/C 카드 (Metadata)
*   **폰트 크기**: 전체적인 정보 밀도를 높이기 위해 메타데이터(카테고리, 저자, 조회수 등)는 PC 대비 축소된 규격(`0.62rem`)을 사용합니다.
*   **날짜 표시**: 모바일 한줄 평 카드 등 간결함이 필요한 곳에서는 날짜를 숨깁니다 (`display: none`).

---

## 3. 타이틀 스타일 가이드 (Typography)
*   **기본 여백**: 모든 타이틀(`.sectionTitle`, `.resultsTitle`)은 `margin: 0`, `padding: 0`을 기본으로 합니다.
*   **포인트 컬러**: 특정 키워드(예: '티끌', '평')에 포인트 컬러(`#ff4804`)를 적용할 때는 `span` 태그를 사용합니다.
*   **장식 제거**: 과거에 사용되던 타이틀 하단의 주황색 언더라인(`::after`)은 미니멀한 디자인을 위해 전역적으로 비활성화합니다.

---

## 4. CSS 작성 원칙 (Clean Code)
1.  **제어권 일원화**: 콘텐츠를 감싸는 컨테이너(`.gridSection`, `.magBody`, `.recentReviewsWrapper` 등)에 `margin-top`이나 `margin-bottom`을 직접 부여하지 마세요.
2.  **!important 최소화**: 표준 레이아웃을 방해하는 기존의 `!important` 속성들은 제거되었으며, 신규 작성 시에도 가이드의 계층 구조를 따르는 것을 원칙으로 합니다.
3.  **모바일 우선**: 모바일 전용 레이아웃 분기 시 반드시 `.newMainLayout` 등의 부모 요소에 `flex-direction: column`이 유지되는지 확인하세요.
