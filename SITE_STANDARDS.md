# 사이트 운영 및 권한 가이드 (SITE_STANDARDS)

이 문서는 에이전트가 코드를 작성하거나 수정할 때 반드시 준수해야 하는 사이트의 핵심 기준을 담고 있습니다.

## 1. 기술 스택 및 작업 방식 (Supabase)
- **Backend:** Supabase를 사용합니다.
- **SQL 가이드:** DB 스키마 변경, RLS(Row Level Security) 설정, 함수 생성 등 서버 측 수정이 필요한 경우, 코드가 아닌 **Supabase SQL 에디터에서 실행할 수 있는 SQL 스크립트**를 별도로 정리하여 제공합니다.

## 2. 권한 체계 (Roles & Permissions)
사용자의 등급에 따라 다음과 같이 권한을 엄격히 제한합니다.

| 등급 | 주요 권한 |
| :--- | :--- |
| **초기 일반 (users)** | 게시물 읽기, 댓글 읽기, 댓글 쓰기 |
| **에디터 (editor)** | (일반 권한 포함) + 자기 닉네임으로 게시물 작성, 본인 게시물 수정/삭제 |
| **관리자 (admin)** | (에디터 권한 포함) + 모든 게시물 수정/삭제, 메인 화면 히어로 카드(메인 노출) 선정 권한 |

## 3. 게시판 운영 구조
게시판은 성격에 따라 3가지로 분류되며, 각기 다른 접근 권한을 가집니다.

### 1) 공지사항 (Notice)
- **작성 권한:** 오직 **관리자(admin)**만 작성 가능합니다.
- **용도:** 사이트 전체 공지 및 중요 안내 사항.

### 2) 매거진 (Magazine)
- **작성 권한:** **에디터(editor)** 이상 작성 가능합니다.
- **필터링 기능:** 
  - 게시물 작성 시 카테고리 설정 필수.
  - 사이드바에서 카테고리 또는 작가 선택 시 자동 필터링 (해당 글만 노출).
  - '모든 글' 선택 시 전체 매거진 게시물 노출.

### 3) 리뷰요청 (Review Request)
- **구조:** 각 작가의 프로필 페이지와 연동됩니다.
- **노출 기준:** 특정 작가의 페이지에서는 **해당 작가에게 요청된 글만** 작성 및 노출되어야 합니다.
- **용도:** 작가와 독자 간의 1:1 소통 및 리뷰 요청.
